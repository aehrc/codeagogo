# ecl-lsp 1.1.0 Integration

Update Codeagogo to use ecl-lsp v1.1.0 packages, adding a simplify hotkey, inactive concept replacement, and updated ECL workbench.

## 1. Package & Bundle Updates

### npm packages

Update `scripts/package.json`:
- `@aehrc/ecl-core` from `^1.0.1` to `^1.1.0`

Rebuild:
- `ecl-core-bundle.js` via `node scripts/bundle-ecl-core.mjs`
- `ecl-editor.standalone.js` — obtain the 1.1.0 standalone build from the ecl-lsp repo (built by the ecl-editor package's build process, or from the GitHub release artifacts)

### What the workbench gains automatically

- Shared language registration fix (no duplicate hovers/completions across instances)
- Toggle display terms (Shift+Alt+T)
- `renderLineHighlight: 'none'` for cleaner embedded appearance
- All ecl-core 1.1.0 improvements flow into code actions and formatting

## 2. ECLBridge Updates

### FormattingOptions

Add `removeRedundantParentheses: Bool = false` to the `FormattingOptions` struct and its `jsObject` computed property. The default remains `false` so the existing format toggle is unchanged.

### New functions

```swift
/// Returns the canonical form of an ECL expression.
/// Strips display terms, sorts AND/OR operands and refinement attributes,
/// flattens same-operator compounds, removes redundant parentheses.
func canonicalise(_ ecl: String) -> String?

/// Compares two ECL expressions structurally.
/// Returns "identical", "structurally_equivalent", or "different".
func compareExpressions(_ a: String, _ b: String) -> String?
```

Both are purely syntactic — no FHIR calls, work in JSContext.

## 3. Simplify ECL Hotkey

### Hotkey

Derived from the ECL Format hotkey by adding Shift. Not independently configurable — when the user changes the format hotkey (e.g. to ⌃⌥F), the simplify hotkey automatically becomes ⇧⌃⌥F.

- New `GlobalHotKey` instance: `eclSimplifyHotKey`, id=7
- Registered with `ECLFormatHotKeySettings.currentModifiers | .shift`, same key code
- When format hotkey settings change (existing Combine pipeline), simplify hotkey updates too

### Menu item

"Simplify ECL" — inserted between "Format ECL" and "Evaluate ECL..." in the status bar menu.

### `simplifyECLSelection()` method on AppDelegate

1. Read selection via `SystemSelectionReader`
2. Parse to check for errors — show error in HUD if invalid, same as format flow
3. Capture any ambiguous-precedence warnings
4. Format with `FormattingOptions(removeRedundantParentheses: true)` — one-way, not a toggle
5. If output matches input (nothing changed): show "No simplification needed" in HUD
6. Replace selection via paste
7. Show "Simplified" in HUD. If there were precedence warnings, show those after.

## 4. Replace Inactive Concepts

### New FHIR operation on OntoserverClient

```swift
struct HistoricalAssociation {
    enum AssociationType: String {
        case replacedBy = "replaced-by"
        case sameAs = "same-as"
        case possiblyEquivalentTo = "possibly-equivalent-to"
        case alternative = "alternative"
    }

    let type: AssociationType
    let targets: [(code: String, display: String)]
}

func getHistoricalAssociations(conceptId: String) async throws -> [HistoricalAssociation]
```

Implementation: `ConceptMap/$translate` with:
- `code` = the inactive concept ID
- `system` = `http://snomed.info/sct`
- `targetsystem` = `http://snomed.info/sct`

Parse the FHIR Parameters response to extract association types and target concepts from the `match` groups, using the `equivalence` field to determine association type.

### Hotkey & settings

New `ECLReplaceInactiveHotKeySettings`:
- Default: Control+Option+I (kVK_ANSI_I = 0x22 = 34)
- Configurable in Settings UI alongside other hotkeys
- Hotkey id=8

### Menu item

"Replace Inactive" — inserted after "Simplify ECL" in the ECL section of the menu.

### `replaceInactiveConceptsInSelection()` method on AppDelegate

1. Read selection via `SystemSelectionReader`
2. Extract concept IDs via `eclBridge.extractConceptIds()`
3. If no concepts found: beep, show "No concepts found" in HUD
4. Batch lookup via `OntoserverClient.batchLookup()` to identify inactive concepts
5. If none are inactive: show "No inactive concepts" in HUD
6. For each inactive concept, call `getHistoricalAssociations()` in parallel
7. For each inactive concept, select the best replacement using priority order:
   - REPLACED BY (1:1 direct replacement)
   - SAME AS (1:1 direct replacement)
   - POSSIBLY EQUIVALENT TO (may be multi-target → OR disjunction)
   - ALTERNATIVE (may be multi-target → OR disjunction)
8. Build replacement text:
   - Single target: `targetCode |targetDisplay|`
   - Multiple targets: `(target1 |display1| OR target2 |display2|)`
   - No targets available: skip (leave original), add to warning list
9. String-replace each inactive concept ID (+ any existing display term `|...|`) with the replacement text, working backwards through the string to preserve offsets
10. Replace selection via paste
11. HUD: "Replaced N inactive concepts" — if any had no replacements available, append warning

### Edge cases

- **Concept has no historical associations**: Leave unchanged, warn in HUD
- **Multiple association types available**: Use priority order above, pick first available
- **Non-SNOMED concept IDs in the selection**: Skip (only SNOMED SCTIDs get historical association lookups)
- **Mixed ECL with operators**: Works — only concept IDs are replaced, surrounding operators/refinements preserved
- **Already-active concepts**: Skip (batch lookup filters these out)

## 5. Menu Item Hotkey Display

All menu items with global hotkeys should show the actual hotkey shortcut, italicised and right-aligned. This applies to all existing items (Lookup, Search, Replace, Format, Evaluate, Shrimp) plus the two new ones (Simplify, Replace Inactive).

### Approach

Use `NSAttributedString` for the menu item `attributedTitle`:
- Title text: system font, left-aligned
- Hotkey text: system font italic, right-aligned via tab stop

```swift
// Example for "Format ECL    ⌃⌥E"
let title = "Format ECL"
let hotkey = eclFormatHotKeySettings.hotkeyDescription  // "⌃⌥E"
let fullText = "\(title)\t\(hotkey)"

let paragraphStyle = NSMutableParagraphStyle()
paragraphStyle.tabStops = [NSTextTab(textAlignment: .right, location: 200)]

let attributed = NSMutableAttributedString(string: fullText, attributes: [
    .font: NSFont.menuFont(ofSize: 0),
    .paragraphStyle: paragraphStyle,
])
// Italicise the hotkey portion
let hotkeyRange = NSRange(location: fullText.count - hotkey.count, length: hotkey.count)
attributed.addAttribute(.font, value: NSFont.systemFont(ofSize: NSFont.systemFontSize, weight: .regular).italic(), range: hotkeyRange)
```

Menu items must update when hotkey settings change. Add a `rebuildMenuItems()` method called from each hotkey settings Combine pipeline.
